Web Historians

masters project

Portfolio > Web Historians

This project was produced in a small team as part of a Masters of Interaction Design at The University of Queensland.

Web Historians is a web app that highlights the importance of archived websites through an interactive educational experience. Targeted at high school IT students who are beginners in HTML, it allows users to update old website code to current web standards.

Users can create accounts, choose a website to restore, edit and validate HTML, save progress and view which modules of which websites they have worked on.

I formed part of the small team that developed this fully-functional app during the course of one semeseter.

My role in delivering this project

Discovering user needs

I conducted user research with high school teachers and background research into high school IT curriculum to understand how this app could fit into the classroom and what the students needs are. As a result of this research, we were able to refine the age group of our target audience and where it could fit into the curriculum as a tool for teachers.

Defining the concept

The concept and specific app features were refined through a process of taking the research results, presentating possible ideas and iterating feedback. This process helped us define which technologies we would use and the specifics of our target audience.

Planning sketch of user interaction through app

App development

  • Explored using HTML imports for repeatable items (e.g. navigation)
  • Set-up Bootstrap using SASS and conducted group training on how to use SASS
  • Set-up HTML template with bootstrap for all pages
  • Created CSS styles for pages
  • Used the Trove API calls to retrieve images for categories
  • Implemented AngularJS to populate websites information on home page
  • Added Javascript to home page to add images from Trove
  • Used PHP, mySQL and jQuery to dynamically add completed modules to user’s ‘My restorations’ page
  • Added dynamic content to pages including breadcrumbs, h1 name and trove link on ‘Working panel’ page
  • Created walkthrough tutorial to ‘Working panel’ page to guide new users through how they can interact with this page

Testing the app

I also ran a digital prototyping session to stress test the login function, test workflow, how users access help, and what they expect on the working panel.

Project management

In addition to my contribution to building the app, I was also the project manager. My role consisted of:

  • writing the project proposal including audience needs and outcomes, risk analysis, task analysis, and a communication plan;
  • coordinating weekly team meetings to check progress, review work to date, talk through any problem areas, and assign new tasks for the week ahead;
  • defining success criteria, writing progress checks, and the final report;
  • conducting the final pitch and demonstration of app in a student tradeshow.
